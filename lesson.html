Alarm: About Alarms

Alarms are created in the software to alert potential problems. There are several different types of alarms varying in severity. Generally, any point generating an alarm will be displayed with a red background. Each alarm will have configurable settings. Alarms can be viewed and acknowledged in the Alarm Console. If your system has internet access, then alarms can be set up with email notification.

Alarm Console

There are two Alarm consuls on your system. The first is located on the home page. This console only displays critical and warning alarms. It is located on the home page to alert you of potential major problems when first logging into in. The second is located on the alarm page. This console displays all alarms on the system. To navigate to these pagesLMBclick the home icon or alarm icon in the quick navigation bar(Fig 1)

Fig.1 -Navigation to Home Page and Alarm Page

Alarm: Configuring Alarms

Finding The Alarm Settings

Alarm Settings can be found on any Unit Settings tab located in the top left of the interface(Fig 1). The settings for alarms are below the notes section of this page. You may need to scroll down to locate this section

Fig.1 -Unit Settings Tab

About Settings

The amount of settings an alarm can have varies from one alarm type to another. All possible settings include: Enable/Disable, Alarm Delay, Alarm Level, Alarm Active, and High / Low Limit(Fig 2). See below for more information about these settings

Fig.2 -Alarm Configuration

Enabled / Disable

A disabled alarm will not generate an email, it will note create an alarm in the alarm counsel, and it will not display red on the system.

Alarm Delay

There are three levels an alarm can have: Critical, Alarm, and Warning

Critical:
These alarms display on the home page and the alarm page. It will also generate an email if your system is configured for alarm email notification. Critical alarms are the only alarm level that will send an email.
Alarm:
These alarms display on the hope page and the alarm page
Warning:
These alarms only display on the alarm console page. Most alarms in your system will be set to a warring level by default.

High and Low Limit

This setting is for alarms that monitor levels such as air temperature or CO2 levels. An alarm is generated if the monitored level rises above the High Limit value. Likewise, it generates an alarm if the monitored level drops below the Low Limit value. The high and low limits can each be disabled independently if you want to just alarm the high or low limit.

Alarm Active

This setting is only for Zone Temperature Alarms. In Occupied mode, an alarm can only generate if the unit is occupied. When switched to the Always position, an alarm can generate when occupied and unoccupied.

Alarm: Using the Alarm Console

About Alarm The Console

There are two Alarm Console’s in your system. The first is on your home page and only displays alarms set to the critical or warring level. The other is located on the Alarm Console page (Fig 1). This Console displays all alarms in the building. Both work in the same.

Fig.1 -Quick Navigation to Consoles

When the system generates an alarm, it is placed in one or both alarm consoles and the Quick Nav Icon for the Alarm Console page turns red. This icon will remain red until all alarms have been acknowledged (Fig 2).

Fig.2 -Unacknowledged Alarm Alert

Alarm Status

Unacknowledged:
Alarms that are unacknowledged are displayed in red
Acknowledged:
Alarms that have been acknowledged will be black
Normal Status:
Alarms that have returned to normal will be green

If an unacknowledged alarm (red) is selected when the acknowledge button is pressed, the alarm will change to acknowledged status (black) but not be removed from the console. If the conditions that created an alarm cease, then the alarm will change to normal status (green) and will need to be cleared from the console. To clear, select the checkbox next to all normal alarms (green). Then LMBclick the acknowledge button (Fig 3).

Schedule: About Occupancy Schedules

About Schedules

Occupancy schedules are one way that your building can lower heating and cooling costs. A common misunderstanding of the Occupied / Unoccupied schedule is it turns your systems on or off. The systems are always on, but during unoccupied hours the equipment works less to maintain a larger range of setpoints.

Note: Some buildings may need an early occupancy for warming up or cooling down before operation hours.

Lead Systems & Subsystems

Subsystems do not have an occupancy schedule, instead they follow their lead systems occupancy schedule. For example, VAVs can only maintain occupied setpoints when their Air Handler Unit (AHU) is occupied. So all VAVs match the occupancy of their AHU (Fig 1).

Fig.2 -Lead & Subsystems Unoccupancy Relation

If a unit has Sub Systems they will have a Subsystems Tab found on the top left of the interface of all lead systems. Not all equipment are Lead Systems.

Schedule: Event Schedules

About Event Schedules

Event schedules can create recurring or one time occupancy schedule changes based on defined criteria. Every schedule can create an event and there is also a building event schedule that effects every system. Event schedules are often used for Holidays, special events, and short term changes to building occupancy.

The event schedules take higher priority over the weekly schedule. Multiple events can be created. If multiple events overlap the highest event on the list will take priority. Priority can be adjust using the up and down arrows orRMB clicking an event and selecting “Move Up” or “Move Down”(Fig 1)

Any changes to events must be saved by pressing the save button or they will be lost.

Settings: About Heating Cooling Setpoints

Unit Modes

A system with an adjustable thermostat operates both in unoccupied or occupied state, but has different heating and cooling setpoints for each occupied state. Either way, the current setpoints the unit is operating on are displayed as “Effective Cooling” and “Effective Heating” on the graphic page. (Fig 1)

Fig.1 -Graphic Tab Effective Cooling & Heating

Based on the zone temperature the unit will have one of three states: Cooling, Heating, and Satisifed.

Cooling Mode:
If the zone temperature rises above the cooling setpoint the unit switches to cooling mode.
Heating Mode:
If the zone temperature drops below the heating setpoint the unit switches to heating mode.
Satisfied Mode:
If the zone temperature is in-between the Cooling and Heating SP then the zone is considered satisfied and the box maintains its minimum setpoints. This temperature range is called the Deadband.

Unoccupied / Occupied

When unoccupied, a box will work to maintain its unoccupied setpoints which can only be configured on this interface. In unoccupied mode, the thermostat thumbwheel cannot adjust the setpoints, but the thermostat does have a temporary occupancy button that will occupy its lead system along with that lead system's subsystems for a duration.

The Heating and Cooling Setpoints are found on the Heating and Cooling Settings tab located in the top left corner of the interface. Only units with adjustable thermostats will have this Tab. If a unit without an adjustable thermostat has heating and cooling setpoints it will be found on the Unit Settings tab located in the top left corner of the interface. (Fig 2)

Fig.2 -Heating / Cooloing & Unit Settings

Settings: Configuring adjustable Heating and Cooling Setpoints

The Interface

A unit with an adjustable thermostat has three settings that control the Occupied Cooling and Heating setpoints: Base SP, Thumwheel Min / Max Adjust (+/-), and Deadband (+/-) (Fig 1)

There is a mathematical breakdown on how the heating and cooling setpoints are calculated. (Fig 2)

(Fig 1) - Occupied Cooling & Heating Settings
(Fig 2) - Graphic Tab Effective & Heating

Base Setpoint

This is the middle value of the Minimum and Maximum Thumbwheel positions. So, if the thermostat thumbwheel is turned to the middle position it would equal the Base Setpoint.

Thumbwheel Min / Max

The Thumbwheel Min / Max represents the highest and lowest value the Thermostat thumbwheel can adjust the base setpoint. The system adds this value to the Base Setpoint to get the Maximum Thumbwheel Adjust Value. Likewise, it subtracts the same value from the Base Setpoint to get the Minimum Thumbwheel Adjust Value. (Fig 3)

Note: The thumbwheel adjust can be disabled and will ignore any position the thermostat may be turned to.

Fig.3 -Thumbwheel Min / Max

Deadband

The Deadband is a temperature range where the system is considered satisfied. The system takes the Adjusted Base Setpoint and adds the Deadband value to get the Occupied Cooling Setpoint. Likewise, it subtracts the deadband value from the Adjusted Base Setpoint to get the Occupied Heating Setpoint of temperatures. (Fig 4) A large Deadband will result in more energy saving but have a zone with a larger range of temperatures.

Fig.4 -Deadband

Trends: About the History Chart Builder

About the History Chart Builder

The History Chart Builder tool allows you to customize what data is displayed on a History Chart. This allows you to visually compare different data.

Setting up the table

First select the time range you wish to view. There are preset time ranges found in the dropdown menu or you can clock icon to manually define the time range. Below these options are settings that give the chart a name or modify how it displays data. (Fig 1)

Fig.1 -History Chart Settings

Below these options are two sections, “Histories” and “Current Charts”. Once you are ready to view the chart then scroll to the bottom of the page and click the “Build Button”.

    Histories:
    This is a list of every history in the system.
    Current Chart:
    This is a list of histories that will be displayed on the built chart.

To add a history to the Current Chart Section LMB click the History. It should populate on the Current Charts. Once it is there, you can change its settings. (Fig 2)

Fig.2 -Current Chart Settings
Color:
Define its color LMB clicking the colored box next. Select the color and click "OK".
Chart Type:
There are different display modes for data that include: Line Chart, Area Chart, Bar Chart, Stacked Bar Chart, Discrete Line Chart, Discrete Area Chart, and Pie Chart. Experiment with the different charts to see what works best.
Up & Down Button:
These move the trend up or down on the list and changes the order it will be displayed on the chart.
The "X" Button:
This removes the History from the Current Charts.
